2) API shape: /api/explorer merges both

Your Explorer page keeps doing:
	•	GET /api/explorer → returns merged list (Anytype + djzsx)

Implementation strategy that actually works with Anytype:
	•	A local worker runs on the user machine (where Anytype vault lives) and periodically syncs Anytype objects → your backend index.
	•	Your backend separately ingests djzsx articles → same index.
	•	Explorer UI reads from the backend index (fast, searchable).

This avoids trying to “read Anytype from the browser” (not reliable).

3) Minimal endpoints you need

Backend (Replit)

Local worker (user machine)

5) Zone mapping rule (works for both sources)

You want consistent Zones, so use one mapping function:
	•	If Anytype object has Zone Code property → use it.
	•	For djzsx articles:
	•	If URL path contains /zones/01-dyor etc → map directly
	•	Else map by tags/keywords (fallback)

Example fallback mapping:

function mapToZoneFromText(text: string) {
  const t = text.toLowerCase();
  if (t.includes("identity") || t.includes("ens") || t.includes("did")) return { code:"02_DID", name:"Decentralized iD" };
  if (t.includes("testnet") || t.includes("sepolia") || t.includes("devnet")) return { code:"03_TESTNET", name:"Blockchain Testnet" };
  if (t.includes("defi") || t.includes("dex") || t.includes("lending")) return { code:"07_DEFI", name:"DeFi" };
  // default:
  return { code:"01_DYOR", name:"DYOR" };
}
